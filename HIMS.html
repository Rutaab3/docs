Problem Definition

Problem Definition

Project Purpose

The Health Insurance Management System is an online application for an organization that wants to manage employee health insurance digitally. The system stores employee information, insurance company records, policy details, employee policy requests, bill approvals, payment processing, and status tracking in one role-based web application.

The earlier process depended on manual records, spreadsheets, scattered software, and repeated follow-up between employees, administrators, managers, and finance staff. This made the workflow time consuming, less secure, difficult to track, and hard to report accurately.

Existing Issues

  • Manual work creates delays and duplicate effort.
  • Employee and policy information is difficult to maintain centrally.
  • Request and bill status is not visible to each role in real time.
  • Security is weak without proper authentication and role-based access.

Proposed Solution

  • Provide a secure login-based system for Admin, Employee, Manager, and Finance Manager.
  • Maintain employee, company, policy, request, approval, and bill data in a database.
  • Support policy search, request submission, approval, payment, and closure workflows.
  • Generate reports and maintain notification logs for important workflow events.

Expected Benefit

  • Reduces manual work and repeated communication.
  • Improves data accuracy, security, and traceability.
  • Makes policy and bill status easier to monitor.
  • Gives each role a focused dashboard and workflow.

CRS

Customer Requirement Specification

Admin Requirements

  • Register, update, activate, deactivate, and manage employees.
  • Add and maintain insurance company records.
  • Add, update, activate, deactivate, and manage policies.
  • Assign policies to employees and review policy requests.
  • Generate reports for employee, request, bill, and date-range activity.

Employee Requirements

  • Login securely using username and password.
  • View and update personal details.
  • Change password using a protected workflow.
  • Search available policies by requirement.
  • Submit policy requests with bill amount and reason.
  • Track request status from pending to closed or rejected.

Manager Requirements

  • View employee insurance requests and generated bills.
  • Review request details before making a decision.
  • Approve or reject requests with remarks.
  • Forward approved bills to the Finance Manager.
  • Maintain profile information through the role dashboard.

Finance Manager Requirements

  • Receive bills forwarded by managers.
  • Review bill amount, request details, and policy information.
  • Credit payment for accepted bills.
  • Close requests after payment is completed.
  • Maintain profile information through the role dashboard.

System Requirements

  • Authenticate users and derive role access on the server side.
  • Maintain status changes for policy requests, approvals, and bills.
  • Store password reset tokens securely and expire them after use or timeout.
  • Log notification events and delivery attempts.
  • Validate important form fields before saving data.

Reporting Requirements

  • Filter reports by date or date range.
  • Display employee, request, and bill records clearly.
  • Support administrative monitoring of workflow progress.
  • Keep output readable in responsive and printable layouts.

Logic

Algorithms

Login Authentication

  1. Accept username and password.
  2. Search admin and staff account stores.
  3. Verify the stored password hash.
  4. Create session values for user id, role, and display name.
  5. Redirect to the role dashboard.

Employee Registration

  1. Admin enters employee profile and password.
  2. Validate name, username, email, phone, role, and date of birth.
  3. Hash password before saving.
  4. Create employee record with active status.
  5. Notify related staff if configured.

Policy Creation

  1. Admin selects insurance company.
  2. Enter policy name, type, premium, coverage, and duration.
  3. Validate numeric and required fields.
  4. Save policy linked to the company.
  5. Show policy in search and management tables.

Policy Request

  1. Employee searches available policies.
  2. Employee opens details and submits request reason and bill amount.
  3. System creates a pending request.
  4. System creates or prepares bill workflow data.
  5. Manager queue receives the request.

Manager Approval

  1. Manager opens pending request details.
  2. Manager approves or rejects with remarks.
  3. Approval record stores manager decision.
  4. Approved request can be forwarded to finance.
  5. Rejected request returns final rejected status.

Finance Payment

  1. Finance Manager reviews forwarded bills.
  2. Finance validates amount and request context.
  3. Finance credits payment.
  4. System marks bill paid.
  5. Finance closes the request after payment.

Password Reset

  1. User enters registered email.
  2. System identifies admin or staff account.
  3. Create secure hashed reset token.
  4. Email reset link to the registered recipient.
  5. Validate token before allowing password update.

Report Generation

  1. Admin selects report filters and date range.
  2. System validates date inputs.
  3. Query matching employees, requests, or bills.
  4. Display results in readable tables.
  5. Show empty state if no records match.
