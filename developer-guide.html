Architecture

Technical Overview

The application is an ASP.NET Core MVC 8 system using a classic controller-view-model structure. Entity Framework Core maps C# models to the Microsoft SQL Server database. Controllers enforce session role checks before allowing protected actions.

The system includes role dashboards, maintenance pages, policy request processing, billing, finance payment, password reset, email notification logging, public content pages, and date-range reports.

Code Organization

Project Structure

Main Folders

  • Controllers contains MVC controller workflow logic.
  • Models contains database entities and view models.
  • Data contains ApplicationDbContext and initializer logic.
  • Services contains SMTP and notification abstractions.
  • Views contains Razor views for public and role pages.
  • wwwroot contains CSS, JavaScript, and images.

Important Files

  • Program.cs registers services, sessions, EF Core, and routing.
  • ApplicationDbContext.cs defines tables, indexes, and relationships.
  • _Layout.cshtml defines shared navigation, sidebar, footer, scripts, and CSS.
  • site.css defines the Material-style teal UI system.
  • app-form-validation.js handles shared client validation behavior.

Environment

Configuration

Connection String

The project requires Microsoft SQL Server or SQL Server Express. The SQL Server connection string from the README is:

Server=.\SQLEXPRESS;Database=HealthInsuranceManagementDB;Trusted_Connection=True;TrustServerCertificate=True;

SMTP Settings

SMTP settings live under the Smtp section in appsettings.json. They are used by the email sender service for reset links and workflow notifications.

Developer Note

Do not commit real production SMTP passwords. For deployment, move secrets to environment variables, user secrets, or secure hosting configuration.

MVC Layer

Controller Responsibilities

HomeController

Public pages including Home, About, Contact, Privacy, Terms, Sitemap, and error pages.

AuthController

Login, logout, forgot password, reset password, and session setup.

AdminController

Companies, policies, employees, assignments, search, and policy request processing.

EmployeeController

Dashboard, details, update details, policy search, policy detail, order insurance, and change password.

ManagerController

Request review, decision, forwarding bills to finance, and profile updates.

FinanceController

Forwarded bill review, payment credit, closing requests, and profile updates.

ReportController

Date-range reports for employees, requests, and bills.

Services

Services and Security

Email Services

  • IEmailSender defines the email send contract.
  • SmtpEmailSender sends SMTP emails through MailKit.
  • INotificationService defines workflow notification methods.
  • EmailNotificationService logs notification attempts.

Security Practices

  • Passwords are hashed using BCrypt.
  • Password reset tokens are stored as hashes.
  • Sessions store user id, username, and role.
  • Protected actions check the current session role.

Validation and UI Assets

Validation

Server-side validation uses data annotations and custom validation attributes. Client-side feedback uses shared JavaScript and field-level messages.

Assets

Page CSS files are stored in wwwroot/css/pages. Shared scripts live in wwwroot/js. The shared theme is in wwwroot/css/site.css.
