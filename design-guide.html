Overview

Design Purpose

The project design is built for an operational healthcare insurance system. It prioritizes clarity, trust, quick scanning, readable forms, dashboard summaries, and role-specific workflows.

The UI follows a calm Material-style direction with teal healthcare colors, white card surfaces, compact navigation, icon-led controls, status pills, and responsive layouts that remain usable on desktop, tablet, and mobile screens.

Brand System

Visual Identity

Brand Mood

Professional, medical, secure, structured, and calm.

Shape Language

Cards and controls use 8px radius. Pills are used only for statuses and compact metadata.

Icon Style

Bootstrap Icons communicate dashboard, profile, policy, search, payment, and logout actions.

Component Standards

Dashboard Hero

Shows role title, context, date/time, and key action buttons.

Metric Card

Shows totals for employees, policies, bills, requests, or status counts.

Status Pill

Used for Pending, Approved, Forwarded, Paid, Closed, Rejected, Active, and Inactive states.

Action Button

Uses icons and concise labels for add, edit, delete, approve, reject, pay, close, and search.

Data Table

Uses readable headings, compact rows, wrapped content, and grouped workflow actions.

Footer Panel

Uses the shared footer background image with a glass-style inner panel.

Input Design

Forms and Validation

Form Standards

  • Use labels and icon-led input groups.
  • Keep forms visually centered and wide enough for professional reading.
  • Use clear primary and secondary button hierarchy.
  • Use password eye toggles on password entry screens.

Validation Standards

  • Show field-level errors below the input.
  • Show success state when a value is valid.
  • Show warning text for password-sensitive flows.
  • Use SweetAlert2 for final success, error, and confirmation messages.

Adaptation

Responsive and Print Design

Desktop

Role pages use sidebar navigation, wide dashboards, multi-column metric cards, and full tables.

Tablet and Mobile

Sidebar becomes compact bottom navigation and content stacks into single-column layouts.

Print

Documentation pages use A4 sizing, page breaks, fixed table layout, and text wrapping for long labels.
